# 四阶段搜索工作流

## 一句话结论

让 LLM 做搜索调研时，用"广搜→追线索→覆盖检查→输出"四阶段工作流，比让 LLM 自由发挥效果好 2-3 倍。

## 场景

任何需要 LLM 通过搜索收集信息的任务：
- 企业/产品调研
- 竞品分析
- 技术选型调研
- 市场研究

## 详细说明

### 问题：LLM 搜索的三大通病

| 通病 | 表现 | 原因 |
|------|------|------|
| **过早停止** | 搜了几次就说"完成了" | LLM 倾向于快速给出答案 |
| **信息密度低** | 只看搜索摘要，不读文章全文 | 默认不会主动 fetch |
| **覆盖偏科** | 某些维度重复搜，其他维度完全遗漏 | 没有覆盖检查机制 |

### 解决方案：四阶段工作流

```
阶段一：广搜
  ├── 英文关键词搜索
  ├── 本地语言关键词搜索
  └── 锚点品牌/竞品搜索

阶段二：追线索
  ├── 挑出排行榜/行业列表文章 URL
  ├── web_fetch 读取全文
  └── 从全文中提取新线索

阶段三：覆盖检查
  ├── 按维度分组已发现的结果
  ├── 标记已覆盖 ✅ 和未覆盖 ❌
  └── 对未覆盖维度针对性补搜

阶段四：去重输出
  ├── 去重合并
  ├── 基本核实
  └── 结构化输出
```

### 每个阶段的关键

**阶段一：广搜**
- 关键词要双语（英文 + 目标市场本地语言）
- 锚点品牌搜竞品："X competitors in Y country"
- 不要急着去重，先尽量多搜

**阶段二：追线索**（最容易被跳过，但价值最高）
- 搜索摘要只有 2-3 行，信息密度低
- 一篇排行榜文章可能列出 20-30 个名字
- **强制要求**：至少 fetch 3 篇文章
- 选文章标准：标题含 "top"、"best"、"list"、"ranking"、"순위" 等

**阶段三：覆盖检查**
- 预定义维度清单（如按子品类、按规模、按地区）
- 要求 LLM 显式列出每个维度的覆盖情况
- 对空白维度用本地语言针对性补搜

**阶段四：去重输出**
- 域名去重 + 名称去重
- 基本核实（至少有搜索结果佐证）
- 结构化 JSON 输出

### Prompt 模板

```
你必须按四个阶段执行搜索：

### 阶段一：广搜
用以下关键词逐一搜索：[关键词列表]
用锚点品牌搜竞品：[品牌列表]

### 阶段二：追线索
从搜索结果中挑出 3-5 篇排行榜/行业列表文章，
用 web_fetch 读取全文，提取更多线索。
**至少 3 次 web_fetch，不能跳过这一步。**

### 阶段三：覆盖检查
将已发现的结果按以下维度分组：
- [维度1]
- [维度2]
- ...
对缺少结果的维度，针对性补搜。

### 阶段四：输出
去重合并，输出 JSON。目标 [N-M] 个结果。
少于 [X] 个说明搜得不够，需要补搜。
```

## 数据支撑

| | 无工作流 | 有四阶段工作流 |
|--|---------|-------------|
| Qwen3-Max | 9 个结果 | 22 个结果（+144%） |
| Claude | 13 个结果 | 21 个结果（+62%） |
| web_fetch 次数 | 0-1 次 | 3-7 次 |
| 品类覆盖 | 3-5/7 | 7/7 |

## 适用范围

- **适用于**：任何需要通过搜索全面覆盖某个领域的任务
- **可能过重**：只需要查一个具体事实（"X 公司的 CEO 是谁"）的简单查询

## 变体

- **技术选型调研**：维度改为"语言/框架/工具/社区活跃度"
- **竞品分析**：维度改为"直接竞品/间接竞品/潜在进入者"
- **市场研究**：维度改为"按地区/按规模/按业态"
