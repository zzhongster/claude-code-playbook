# 设计期强模型、运行期廉价模型 — 把判断力编译进提示词与结构

> 来源：trade-ai 深度报告优化实战（2026-07-16~17，#328-335 八个 Issue 全程）
> 实证：生产 Opus ¥46/33min → glm-5.2 v4p2 ¥9-13/13-22min（成本 -73~-80%，耗时 -35~-60%），事实可靠性反升

## 核心理念

LLM 应用降本的常见死路是"直接换便宜模型然后接受质量下降"。正确的打开方式：

**强模型（Fable/Opus 级）的智力只花在设计期**——写更好的提示词、设计更强的约束结构、编排更聪明的调度代码；**运行期全部用廉价模型**（glm-5.2/qwen 级）。判断力被"编译"进提示词与结构，而不是在运行时按 token 购买判断力。

## 实战中的四次"编译"

### 1. 把领域视角编译进 system prompt
廉价模型不知道"报告读者是谁"，会写出面面俱到但没有立场的内容。设计期把视角锚定写死：
> "本报告的读者是中国供应商……与此无关的信息再有趣也不写；与此相关的风险再敏感也不能漏。"

### 2. 把可靠性纪律编译成结构性约束（最关键）
廉价模型会自信地断言"该公司完全不覆盖宠物品类"（实际官网在售，三轮重跑全错）。修法不是"提醒它仔细"，而是**结构性防翻车**：
- schema 层：Step 1 强制产出「官方面清单」（官网导航/品牌/在售品类枚举），下游把它当 ground truth 查表
- prompt 层：绝对化断言（没有/空白/唯一/从未）必须先做反例检索，记录写入 `counter_checks` 字段，否则输出无效
- 代码层：纯正则微审计（零 LLM 成本）扫描产出，违规回喂做单轮修正

### 3. 把"预算纪律"编译进提示词
廉价模型会把轮数预算全部花完（给它省下的轮数，它会拿去跑更多衍生检索）。设计期加抓取纪律：
> "同一域名最多抓 2 个页面；官网事实无冲突时不再为同一事实追加检索；把预算留给还没查清的字段。"

### 4. 把调度智能编译进编排代码
研究步骤的依赖关系（DAG）写进 playbook 元数据，调度器就绪即发（asyncio 数据流 + semaphore），10 步串行 33 分钟 → 并行 13-22 分钟。模型不需要知道自己在并行。

## 收益核算（三家公司真实重放 A/B）

| 阶段 | 成本 | 耗时 | 关键质量事件 |
|------|------|------|-------------|
| Opus 直跑 | ¥46 | 33min | 事实基本可靠但无护栏 |
| 换 glm（裸） | ¥10 | 31min | 品类误判三轮全错、营收虚标无人拦 |
| glm + 四次编译 | ¥9-13 | 13-22min | 品类误判根治、营收虚标被核验闸门抓住、核验全绿出现 |

## 适用判断

- ✅ 高频、流程化、有明确产出 schema 的 LLM 任务（报告/抽取/分类流水线）
- ✅ 廉价模型的失败模式**可枚举**（绝对化断言/占位符/预算挥霍）——可枚举才能编译成约束
- ❌ 开放式创造/一次性任务：编译成本摊不平，直接用强模型
