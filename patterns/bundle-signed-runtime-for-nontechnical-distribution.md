# 模式：给非技术用户分发需运行时的工具，内置官方签名二进制——别要求预装，也别自制未签名 exe

## 一句话结论

当一个要发给不懂技术的人的工具，底层离不开某个运行时（Node、Python、JRE…），有三条路：①要求用户自己预装 ②把运行时和代码编译成一个自制单文件 exe ③**把官方发布的运行时二进制直接打包进分发包，脚本用包内的那份**。对非技术用户，③几乎总是最优：用户零操作（不用装任何东西），且用的是**官方签名二进制**——不会被 Windows Defender/SmartScreen/杀毒拦。①有"装配套工具"的摩擦，②的自制未签名 exe 几乎必被安全软件拦（非技术用户看到"已保护你的电脑"就卡死），而你还得交叉编译+真机验证一个你可能没有的目标环境。

## 场景

- 分发对象是不懂技术的同事/客户，目标是"双击即用、零前置"
- 工具本身依赖某语言运行时（如 MCP 客户端插件依赖 Node 跑 mcp-remote 桥接）
- 你想省掉"让用户先装 Node/Python"这一步

## 三条路对比

| 路线 | 用户操作 | 被安全软件拦 | 你的验证成本 | 包体积 |
|---|---|---|---|---|
| ① 要求预装运行时 | 装一次（有摩擦、会卡人） | 否 | 低 | 最小 |
| ② 自制单文件 exe（pkg/SEA 编译） | 零 | **几乎必被拦**（未签名） | 高（交叉编译+真机验证） | 中 |
| ③ **内置官方签名二进制** | 零 | 否（官方签名） | 中（只打包不编译） | 大（+运行时体积） |

③的关键优势：**官方 node.exe / python.exe 是签名的**，SmartScreen/Defender 放行；你只是"复制官方文件"不是"编译新 exe"，没有签名问题、没有交叉编译坑。

## 做法（以 Node/MCP 为例）

1. **下载官方运行时二进制**放进分发包：`vendor/win-x64/node.exe`（从 nodejs.org/dist 直接下，选 LTS，与你验证时的版本一致）。
2. **预打包依赖**：若依赖是纯 JS（`find node_modules -name "*.node"` 为 0，无原生模块），Mac 上装的 `node_modules` 可直接跨平台打进包——完全离线，安装时不联网。有原生模块则不能跨平台预打包，退回"包内运行时 + 安装时装依赖"。
3. **安装脚本复制到固定位置**（`%USERPROFILE%\.trade-data-mcp\`），配置指向复制后的绝对路径——用户删了安装包也不影响。
4. **保留回退兜底**：包内运行时缺失（解压不完整）→ 回退找系统已装的运行时，别硬失败。
5. **诚实标注验证边界**：你若不在目标平台上，无法真机验证执行。能验证的（依赖能连通、二进制是正确的目标平台 PE/ELF、脚本结构自检、包完整）都做，然后明说"需一位目标平台用户首次确认"。

## 坑

- **只需要 node.exe 一个文件**：Windows 的 node.exe 是自包含静态链接的，跑 `node script.js` 不需要 npm/其他文件。但如果安装时要 `npm install`，就得带完整 node 发行版（含 npm）——所以优先预打包依赖（第2步），只带 node.exe。
- **架构**：win-x64 覆盖绝大多数；ARM Windows 极少（Win11 ARM 有 x64 模拟能跑，慢），文档留一句"ARM 架构联系管理员"即可。
- **体积**：Node 二进制 ~88MB，zip 压缩后 ~30MB，仍在微信可传范围。值不值看你多想省用户那一步。
- 脚本本身的跨平台坑照样要防（中文脚本 UTF-8 BOM、别依赖目标机没有的 CLT）——见 `ps51-utf8-no-bom-gbk-mangles-chinese` / `macos-usr-bin-python3-clt-stub`。

## 教训

- "让用户装个 X 就行"对技术人是一句话，对非技术用户是一道坎——每多一个前置步骤，就多一批人卡在那里。能内置就内置。
- 自制 exe 看着最干净（单文件），但**未签名是它的原罪**：给非技术用户分发时，被安全软件拦一次就前功尽弃，而你还没法在自己机器上复现那个拦截。官方签名二进制绕开了整类问题。
- trade-agent-data 插件（2026-07）：Windows 安装包从"要求用户装 Node"改为"内置官方 v24 node.exe + 预打包纯 JS 的 mcp-remote"，包 15KB→35MB，换来真正的双击零依赖。同主题的分发可靠性坑已踩三个（编码 BOM、CLT shim、这个），核心教训一致：**分发给非技术用户，按"刚开箱、什么都没装、还开着杀毒"的机器来设计**。
