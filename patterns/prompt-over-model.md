# Prompt > 模型 > 架构

## 一句话结论

优化 prompt 的效果远大于换更贵的模型，换模型的效果远大于改架构。在 LLM 应用开发中，投入产出比最高的优化方向是 prompt。

## 场景

当你面临以下决策时：
- "效果不好，要不要换个更贵的模型？"
- "Claude 太贵了，换 Qwen 行不行？"
- "要不要把单体改成多 Agent 并行？"

## 详细说明

### 影响力排序（实验数据）

| 优化方向 | 投入 | 效果 | 性价比 |
|---------|------|------|--------|
| **Prompt 优化** | 改几行文字 | +62% ~ +144% | ★★★★★ |
| **模型选择** | 改一个配置 | <5% 差异 | ★★ |
| **架构重构** | 重写代码 | 可能变差 | ★ |

### 为什么 Prompt 优化效果最大

LLM 本质上是"按指令执行的通用引擎"。指令不够具体时，LLM 会：
- 过早停止（"差不多了吧"）
- 遗漏维度（不知道还要查什么）
- 自由发挥（可能偏离目标）

具体化 prompt 等于给 LLM 一份 checklist，消除了上述三类问题。

### 关键的 prompt 优化技巧

1. **给明确的工作流阶段**：不说"请搜索"，而说"先搜索→再读文章→检查覆盖→输出"
2. **给量化目标**：不说"尽量多找"，而说"目标 15-25 个，少于 12 个说明不够"
3. **给覆盖检查清单**：列出应该覆盖的维度，要求逐一检查
4. **强制使用特定工具**："至少 3 次 web_fetch 读文章全文"
5. **明确边界**："规模大小不是淘汰依据"（防止 LLM 过度解读）

### 为什么模型差异不大

现代 LLM（Claude、Qwen3-Max、GPT-4o 等）在遵循结构化指令方面能力接近。差异主要在：
- 训练数据覆盖（Claude 对某些领域的知识更丰富）
- 推理深度（复杂逻辑链 Claude 更稳定）
- 工具调用效率（Claude 平均每次搜索价值更高）

但这些差异被好的 prompt 大幅缩小。

## 适用范围

- **适用于**：搜索、调研、数据提取、分类筛选、报告生成等任务
- **可能不适用**：纯数学推理、代码生成中的复杂算法设计（这些任务模型能力差异更大）

## 行动指南

遇到"效果不好"时，按这个顺序尝试：

```
1. 先优化 prompt（成本：0，耗时：10 分钟）
   → 加工作流阶段、加目标数量、加覆盖清单、加工具使用要求

2. 如果 prompt 优化后仍不够，再考虑换模型（成本：可能更高）
   → 先试便宜模型 + 好 prompt，再试贵模型

3. 最后才考虑改架构（成本：重写代码）
   → 确保不是 prompt 问题伪装成架构问题
```
