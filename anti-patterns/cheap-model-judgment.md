# 反模式：用便宜模型做判断类任务

## 一句话结论

便宜模型在"遵循指令搜索"上表现不错，但在"理解意图做判断"上容易翻车——特别是当 prompt 有模糊空间时。

## 踩坑场景

"筛选/分类只是简单判断，用最便宜的模型就行"

## 发生了什么

用 MiniMax M2.5（~$0.10/MTok）做企业筛选。调研描述写了"中小型日用杂货零售连锁"，筛选标准只定义了"有实体门店 + 门店 ≥ 3"。

MiniMax 的行为：
- 把 description 中的"中小型"当成硬性标准（实际只是目标偏好）
- Daiso（1500 店，$2.9B 营收）→ 淘汰，理由"大型企业规模不符"
- Artbox（190 店，文具杂货）→ 淘汰，理由"文具非杂货"
- 40 家全部淘汰，0 保留

同一任务，Claude（包含在 Max 订阅中）和 Qwen3-Max（$1.20/MTok）都能正确理解"中小型是偏好不是硬性标准"。

## 根因

便宜模型的问题不在于"笨"，而在于**对模糊指令的处理策略不同**：

| 模型 | 遇到模糊指令时的行为 |
|------|-------------------|
| 贵模型（Claude/Qwen3-Max） | 推断意图，选择合理的解释 |
| 便宜模型（MiniMax/Haiku） | 倾向字面执行，宁可错杀不放过 |

筛选/判断任务天然充满模糊空间：
- "相关"还是"不相关"？
- "规模合适"还是"太大"？
- "品类匹配"到什么程度？

这些判断需要理解全局意图，不是简单的规则匹配。

## 适用场景分析

| 任务类型 | 便宜模型能做？ | 原因 |
|---------|-------------|------|
| 结构化数据提取 | ✅ 能 | 规则明确，无模糊空间 |
| 格式转换 | ✅ 能 | 纯机械操作 |
| 简单分类（二选一） | ✅ 能 | 边界清晰 |
| 多维度综合判断 | ❌ 慎用 | 有模糊空间 |
| 理解用户意图的筛选 | ❌ 慎用 | 需要推断 |
| 边界 case 处理 | ❌ 慎用 | 需要常识推理 |

## 替代方案

### 方案一：花钱买准确（推荐）

判断类任务用中档以上模型（Qwen3-Max $1.20/MTok 起）。单次调用 token 量小，实际成本差异很小。

### 方案二：消除模糊空间

如果必须用便宜模型，把 prompt 写得极其明确：

```diff
- "调研中小型日用杂货零售连锁"
+ "调研日用杂货零售连锁。
+  硬性标准仅为：有实体门店 + 门店 ≥ 3。
+  企业规模大小不是淘汰依据。
+  description 中的'中小型'是目标偏好，不是筛选条件。"
```

### 方案三：代码规则 + LLM 兜底

对明确的规则（门店数 ≥ 3）用代码判断，只把边界 case 交给 LLM。
