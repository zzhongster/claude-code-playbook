# 反模式：批量用 pypinyin 生成中文地名英文名——多音字地名会静默转错，且错得像对的

## 一句话结论

用 `pypinyin` 批量把中文城市/地名转成拼音英文名（建归一字典）时，**多音字地名会转错**：库按最常见读音转，但地名往往用非最常见的那个音。`长治`→`ZHANGZHI`（应 CHANGZHI，长在此读 cháng）、`漯河`→`TAHE`（应 LUOHE）、`昌都`→`CHANGDOU`（应 CHANGDU，都读 dū）、`朝阳`→`ZHAOYANG`（应 CHAOYANG，朝读 cháo）。这些错误不报异常、拼出来也像个正常拼音，肉眼扫一遍很难发现——必须逐条人工审多音字 + 用真实数据验证命中。

## 场景

- 建"中文地名→英文/拼音"归一字典（城市/港口检索归一、地址标准化）
- 图省事用 `pypinyin` 对几百个地名批量生成，不逐条核
- 多音字地名（含 长/朝/都/重/六/漯/华 等字）被转成最常见读音，与官方罗马字不符

## 踩坑经历

trade-agent-data（2026-07）建 519 项城市/港口中英归一字典：pypinyin 生成后人工审出 3 个多音字错（长治/漯河/昌都），改对了；但**漏了朝阳→ZHAOYANG**。上线前 reviewer 用真实海关数据交叉验证才抓到——决定性证据：
- `supplier_city="chaoyang"`（正确值）→ 78 家命中，含"朝阳长征轮胎"（辽宁朝阳真实知名企业）。
- `supplier_city="zhaoyang"`（pypinyin 错值）→ 31 家命中，全是"赵阳/昭阳"人名品牌撞词的无关公司，**没一家是朝阳本地企业**。

即：错误的拼音不是"查不到"，而是**系统性查到一批错误的公司**（假阳性）+ 漏掉真正的（假阴性），比零命中更隐蔽。

## 解法

1. **多音字地名清单先过一遍**：含 长/重/朝/都/六/华/单/乐/蚌/柏/涡/藕/汶/浚/邘 等已知多音字的地名，逐条核官方罗马字（民政部/官方英文名）。
2. **民族语/历史译名例外**：哈尔滨 HARBIN、乌鲁木齐 URUMQI、呼和浩特 HOHHOT、拉萨 LHASA、厦门 XIAMEN、香港 HONG KONG、澳门 MACAO——这些不是拼音，pypinyin 也给不对，要手工列。
3. **真实数据验证，不只人工扫读**：把归一值拿去真实数据里查，看命中的是不是该地企业。错值往往命中一批"撞词"的无关记录，一验就露——比肉眼审字典可靠得多。
4. **拿不准宁可不收**：边远/少数民族地区地名（巴彦淖尔/克拉玛依）拼音与民族语译名并存、置信度不足时，保留标准拼音（拼音不算错，只是可能非最常见），别猜测性地改。

## 教训

- `pypinyin` 是解析器不是权威地名库：它按字读音拼，不懂"这个地名习惯读哪个音"。批量生成必须叠加人工审 + 数据验证两道。
- 拼音错误的危险在于**看起来对**：ZHAOYANG 是个合法拼音串，不触发任何告警，只有懂"朝阳读 cháo"或拿数据一验才知道错。这类"静默且像对的"错误，是数据字典里最难抓的一类（同族：多语言名称检索只查一个字段 / strptime 未补零宽松解析）。
- 归一字典的验收标准应是"归一后能在真实数据里命中该实体"，而非"字典条数达标 + 抽样看着对"。
