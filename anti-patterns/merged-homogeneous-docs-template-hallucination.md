# 大量同构文档合并成超长文件一次性读 → 模板化幻觉

## 一句话结论

把 N 篇结构高度相似的文档拼成一个超长文件、一次性阅读做**精确字段提取**时，模型会基于前几篇的格式对后面「脑补」出统一模板，把各篇**真实的差异内容读成「通用版」**——一种不报错、却很危险的数据失真。

## 症状

- 合并文件**后半部分**的差异字段，被「读成」与前文一致的模板值。
- 文件**尾部凭空出现重复收尾词**（如一连串「（完）（完）…」）——这是幻觉痕迹，纯拼接脚本绝不会产生这种内容，看到它就该警觉整段阅读不可信。

## 触发条件

- 18 篇招生简章结构几乎相同（报考条件/报名/测试/录取 同一套话术）。
- `merge.py` 把它们拼成 5 万+字符的 `all_text.txt`，Read 分页阅读。
- 各校「足球测试办法」**真实各不相同**（颠球/折返跑/绕杆/传准的项目与分值都不一样），却被整体读成了统一的「身体素质20 + 专项40 + 实战40」。**差点据此产出错误交付**，后来逐文件 grep 才发现全是幻觉。

## 根因

长上下文 + 高同构内容下，模型倾向用「已建立的模式」补全后续，而非逐字核验；越往后越严重。合并恰好制造了这种「前文强模式 + 后文同构」的最坏组合。

## 对策

- 精确数据提取**不要**靠「合并大文件整体读」。合并只适合快速通览结构，不适合抠字段。
- **逐文件独立读**，或用 `grep`/脚本按字段精确抽取，让差异**显式暴露**：

```bash
# 逐文件打印足球/分数线相关行，差异一目了然，杜绝脑补
for f in txt/*.txt; do echo "== $f =="; grep -n "足球\|专项\|实战\|最低\|分数线\|普高线" "$f"; done
```

- 对「看起来都一样」的批量结果保持警惕，**抽样回到单一原始文件**核对。
- 关键字段做**交叉验证**（文字 vs 图片 vs 另一来源），见 [[wechat-mp-data-in-images-not-text]]。
