# 贡献指南

## 内容分类

提交前先判断你的内容属于哪一类：

| 类型 | 目录 | 要求 |
|------|------|------|
| **Pattern（模式）** | `patterns/` | 至少在 2 个项目中验证过，有明确的适用场景和结论 |
| **Anti-pattern（反模式）** | `anti-patterns/` | 有具体的踩坑经历，说明原因和替代方案 |
| **Experiment（实验）** | `experiments/` | 有对照组、有数据、有结论 |
| **Foundation（基础）** | `foundations/` | 需讨论，提 issue 先 |

## 文档格式

### Pattern / Anti-pattern

```markdown
# 标题

## 一句话结论

用一句话概括核心发现。

## 场景

什么时候会遇到这个问题 / 需要这个模式。

## 详细说明

具体内容。

## 数据支撑（如有）

对比数据、实验结果。

## 适用范围

- 适用于：...
- 不适用于：...

## 相关

- 链接到相关的 pattern / anti-pattern / experiment
```

### Experiment

```markdown
# 实验：标题

**日期**：YYYY-MM-DD
**项目**：在哪个项目上做的实验

## 假设

要验证什么。

## 实验设计

对照组、变量控制。

## 数据

表格形式呈现。

## 结论

数据支持还是否定了假设。

## 可复用的发现

哪些结论可以推广到其他场景。
```

## 提交方式

1. Fork repo
2. 新建分支
3. 按上面的格式写文档
4. 提 PR，说明内容分类和核心结论

如果只是一个想法或初步观察（还没验证），可以先开 issue 讨论。
