# 实验：单对话 vs 多 Worker vs Prompt 优化 — 全量对比

**日期**：2026-03-12
**项目**：企业调研引擎（enterprise-research）

## 假设

1. 多 Worker 并行架构（Pipeline）应该比单对话（Skill）更快更便宜
2. 便宜模型（Qwen）在搜索任务上能替代贵模型（Claude）
3. Prompt 优化的效果可能不如换更好的模型

## 实验设计

同一个调研任务（韩国均一价零售商），4 种方案对比：

| 方案 | 模型 | 架构 | Prompt |
|------|------|------|--------|
| A. Skill 原版 | Claude (Max 订阅) | 单对话 | 旧版（无四阶段工作流） |
| B. Pipeline | Qwen3.5 Plus + MiniMax + Sonnet | 多 Worker 并行 | 各 Worker 独立 prompt |
| C. Qwen3-Max 单对话 v1 | Qwen3-Max | 单对话 | 旧版 |
| D. Qwen3-Max 单对话 v2 | Qwen3-Max | 单对话 | 优化版（四阶段工作流） |
| E. Skill 优化版 | Claude (Max 订阅) | 单对话 | 优化版（四阶段工作流） |

控制变量：搜索关键词（13 个）、锚点品牌（4 个）、调研字段（8 个）、筛选标准完全相同。

### Pipeline 全灭原因分析

Pipeline 的 FilterWorker（MiniMax 模型）把 40 家全部淘汰：
- 12 家因"大型企业规模不符"（把 description 中的"中小型"当硬性标准）
- 11 家因"业态不符"（Artbox 文具、Modern House 家居被误判）
- 8 家因门店数不足
- 其余因已关闭/撤出/信息不足

根因：便宜模型 + 孤立上下文 = 过度解读筛选条件。FilterWorker 只看到 prompt + 数据，没有调研全程的意图理解。

### Prompt 优化的具体内容

v1 → v2 的改动（模型不变）：

1. **强制四阶段工作流**：广搜→追线索→品类覆盖检查→输出（而非 LLM 自由发挥）
2. **要求至少 3 次 web_fetch**：读行业文章全文，不能只靠搜索摘要
3. **品类覆盖检查清单**：列出 7 个子品类，每个都要有企业
4. **明确目标数量**：15-25 家（防止 LLM 过早停止）
5. **明确筛选边界**：企业规模不是淘汰依据

## 结论

### 结论 1：Prompt 优化 > 模型选择 > 架构选择

影响力排序明确：

| 变量 | 对比 | 效果 |
|------|------|------|
| Prompt 优化（同一模型） | C→D (Qwen)：9→22 家 | **+144%** |
| Prompt 优化（同一模型） | A→E (Claude)：13→21 家 | **+62%** |
| 模型选择（同一 prompt） | D vs E：22 vs 21 家 | **<5% 差异** |
| 架构选择 | B vs D/E：0 vs 21-22 家 | Pipeline 完败 |

### 结论 2：单对话 >> 碎片化多 Worker（对搜索/判断类任务）

多 Worker 架构的根本问题：
- 每个 Worker 是孤立对话，没有"还缺什么品类"的全局意识
- SearchWorker 搜索量大（309 次）但方向偏，找到大量不相关企业
- FilterWorker 缺乏全局意图，过度解读筛选条件

单对话的优势：
- 全局上下文，能主动补盲区
- 搜索更精准，30 次搜索 > 309 次搜索
- 筛选更合理，理解意图而非死板执行规则

### 结论 3：四阶段搜索工作流是通用模式

不依赖具体模型或平台，可复用到任何调研/搜索类任务：
1. 广搜（关键词 + 锚点竞品）
2. 追线索（读文章全文提取更多线索）
3. 覆盖检查（按维度检查遗漏，补搜）
4. 去重输出

## 可复用的发现

1. **写 prompt 时**：给 LLM 明确的阶段性工作流，比给"自由发挥"的空间更好
2. **选架构时**：需要全局意识的任务（搜索、判断、筛选）用单对话；独立无依赖的任务（抓取 N 个 URL、处理 N 个文件）才用多 Worker
3. **选模型时**：先优化 prompt，prompt 到位后，便宜模型（Qwen3-Max $1.2/MTok）和贵模型（Claude $15/MTok）效果差异 <5%
4. **做筛选/判断时**：确保判断者有全局上下文，孤立的判断容易过度解读
5. **强制 web_fetch**：搜索摘要信息密度低，读 3 篇行业文章全文抵得上 30 次搜索
